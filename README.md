🏛️ The Montessori Album Factory

Tools for usage are in .md format.

Hosted hub for transmuting AMI Elementary training into HTML lesson modules.

Designed to bypass formatting friction for 2E brains and allow for rapid-fire deployment around rubric requirements. It was built with a gifted friendly workflow to hit aggressive deadlines through "Batch Processing" and "Assembly Line" coding.

🎯 Project Mission

The goal of this project is to take raw transcripts from AMI lectures and instantly "Transmute" them into a cohesive, searchable web experience without sacrificing the structural integrity of the standard Montessori rubric.

The Core Principles

Speed Over Perfectionism: Optimized for a high-volume "Assembly Line" workflow.

The Sacred Rubric: Every lesson strictly adheres to the "Grey/Yellow" layout (Material Box / Aside Box).

Zero-Dependency Core: Built with Vanilla HTML and Tailwind CSS (via CDN) to ensure long-term stability and zero build-step overhead.

🛠️ Architecture

The site operates on a Static Hub & Spoke model:

index.html (The Dashboard): The central command center. It uses a REPO_DATA object to dynamically populate the curriculum map and link to individual lessons.

/lessons/{subject}/{lesson-name}.html: Standalone lesson files. Each file is self-contained, meaning it can be shared, printed, or exported individually.

Tech Stack

Styling: Tailwind CSS (Modern, fluid, responsive).

Typography: Crimson Pro for academic body text; Inter for procedural actions.

Icons: Lucide-React (via CDN).

Illustrations: Inline SVGs using "squiggly" paths to mimic hand-drawn Montessori sketches.

📋 The "Grey/Yellow" Rubric Alignment

Every lesson module generated by this factory follows the AMI standard:

Header Section: Subject, Title, and Guide attribution.

Purpose Section: Clear, concise pedagogical aims.

Materials Needed (Grey Box)

Presentation Procedural: - Bold Actions: What the guide does.

- Italic Dialogue: What the guide says.

Illustrate Divs: Custom SVG placeholders for stick-figure sketches.

Instructor's Aside (Yellow Box): Contextual "why," fun facts, or scientific insights.

🚀 How to Add Lessons

Generate the File: Use the lesson-template.html (or solids-of-rotation.html) as your visual anchor.

Name the File: Use kebab-case-naming.html.

Update the Index: In index.html, locate the REPO_DATA constant and append your new lesson title to the appropriate subject array:

const REPO_DATA = {
  "Music": {
    lessons: ["Tone Bars", "Rhythm", "New Lesson Title"]
  }
};


The system automatically calculates the filepath based on the title.
